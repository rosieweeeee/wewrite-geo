# wewrite-geo 策略说明 & 使用指南

---

## 一、这个 Skill 解决什么问题

### 背景
元宝、DeepSeek、豆包、Kimi 等 AI 大模型已经能直接读取微信公众号内容。
当用户向这些 AI 提问时，AI 会检索并引用公众号文章作为回答依据。

**这意味着：一篇被 AI 频繁引用的公众号文章，等于获得了免费的 AI 推荐流量。**

### 问题
普通公众号文章是为"人类读者"写的——情绪化开头、叙事性结构、追求爆款。
但 AI 引用内容有完全不同的偏好：直接答案、结构清晰、有数据支撑、实体定义明确。

### 解法
wewrite-geo 在保证文章对人类可读的前提下，系统性地注入"AI 喜欢引用"的内容结构，
同时通过内容矩阵规划，让品牌在多类用户问题下都能被 AI 检索到。

---

## 二、核心策略

### 策略一：品牌实体锚定
每篇文章开头必须包含**品牌定义句**（`brand_definition`），格式为：
> "[品牌名] 是一款……，主要解决……"

这句话是 AI 建立品牌认知的锚点，会被 AI 直接截取引用。
定义句越准确、越具体，AI 对品牌的描述就越准确。

### 策略二：问题驱动选题
选题不只追热点，而是**以用户会直接问 AI 的完整问句为核心**：
- 用户问"圆周旅迹怎么用" → 对应文章就会被 AI 在回答这个问题时引用
- 用户问"旅行规划用什么 App" → 对应文章就能出现在 AI 的推荐里

问题来源有三层（优先级从高到低）：
```
用户自己上传的问题（最了解自己用户）
    ↓
从知乎/V2EX 实时采集的真实用户提问
    ↓
系统根据品牌信息自动生成的基础问题
```

### 策略三：内容矩阵覆盖
GEO 不是靠一篇爆款，而是**系统性覆盖 7 类内容**，让 AI 在不同场景下都能找到你：

| 类型 | 覆盖的用户问题场景 | 建议顺序 |
|------|-----------------|---------|
| 品牌定义 | "X 是什么" | 第 1 篇 |
| 场景适配 | "X 适合谁/什么场景用" | 第 2 篇 |
| 使用教程 | "X 怎么用" | 第 3 篇 |
| 深度 Q&A | "X 的常见问题" | 第 4 篇 |
| 竞品对比 | "X 和 Y 哪个好" | 第 5 篇 |
| 用户故事 | "真实用户用 X 的体验" | 第 6 篇 |
| 行业洞察 | "X 所在行业的趋势" | 第 7 篇 |

Skill 会追踪每类是否已有文章覆盖，优先推荐未覆盖的高优先级类型。

### 策略四：Q&A 模块
每篇文章中段必须包含**结构化 Q&A 模块**（5-8 个问答）。
这是元宝/DeepSeek 解析效率最高的内容格式，被引用概率显著高于普通段落。
Q&A 的问题直接来自问题库（优先用户上传的真实问题）。

### 策略五：可引用密度
每篇文章必须包含至少 3 种"AI 可以直接截取引用"的句型：
- **定义句**："X 是指……"
- **数据句**："根据[来源]，[具体数字]……"
- **步骤句**："只需 N 步：第一步……"
- **对比句**："相比[竞品]，X 在[维度]上……"

## 三、完整使用指南

### 3.1 首次使用（必做）

**第一步：配置品牌信息**
```
说：GEO设置
```
按提示填写（约 2 分钟）：
- 品牌名（官方名称）
- 品牌一句话定义（具体、无空话，会被 AI 直接引用）
- 3-5 个核心价值点
- 主要竞品名称

完成后生成 `geo-config.yaml`。

**第二步：建立问题库（强烈建议）**
```
说：更新问题库
```
从知乎/V2EX 自动采集用户真实提问，归类存入 `query-bank.yaml`。

或者直接上传你已知的用户问题：
```
说：上传问题
[粘贴你收集的用户问题列表]
```

**第三步：安装 WeWrite（推送草稿箱需要）**
```bash
git clone --depth 1 https://github.com/oaker-io/wewrite.git ~/.openclaw/skills/wewrite
cd ~/.openclaw/skills/wewrite && pip install -r requirements.txt
cp config.example.yaml config.yaml
# 在 config.yaml 中填入微信公众号 appid 和 secret
```
完成后说"重新检查环境"启用推送功能。

---

### 3.2 日常写作流程

#### 场景 A：全自动（最常用）
```
说：写一篇公众号文章
```
Skill 自动完成：选题 → 框架 → 素材采集 → 写作 → 自检 → 摘要 → 排版 → 推送草稿箱

**适合**：日常内容生产，快速产出

---

#### 场景 B：交互模式（想自己把控选题和框架）
```
说：交互模式，写一篇公众号文章
```
在以下节点暂停等你确认：
- Step 1 结束后：展示 10 个选题，你选一个
- Step 2 开始时：展示推荐框架，你确认或切换

**适合**：对选题有想法，但写作过程还是想 AI 代劳

---

#### 场景 C：指定选题直接写
```
说：用这个选题写：圆周旅迹和马蜂窝哪个更适合自由行规划？
```
或先把选题存起来，下次再写：
```
说：上传选题：[你的选题列表]
```
下次说"写一篇公众号文章"时会优先使用已上传的选题。

**适合**：已经有明确内容计划，或运营团队提前排期

---

### 3.3 问题库管理

问题库是 Q&A 模块的核心数据源，质量越高，文章被 AI 引用的概率越大。

| 操作 | 指令 | 说明 |
|------|------|------|
| 自动采集问题 | `更新问题库` | 从知乎/V2EX 爬取，每月建议更新一次 |
| 手动上传问题 | `上传问题` + 粘贴问题列表 | 用户反馈/销售问题/客服高频问题都可以上传 |
| 查看当前问题 | `看问题库` | 展示所有问题及使用状态 |

**问题来源建议**：
- 客服收到的用户咨询（最真实）
- 销售/运营团队收集的用户问题
- 社群/评论区里的提问
- 竞品的评论区（了解用户关心什么）

---

### 3.4 内容进度管理

```
说：看内容进度
```
展示 7 类内容矩阵的覆盖状态，告诉你还缺哪类内容。

建议的写作节奏（以圆周旅迹为例）：

| 周次 | 推荐内容类型 | 示例选题 |
|------|------------|---------|
| 第 1 周 | 品牌定义 | "圆周旅迹是什么？和携程马蜂窝有什么区别" |
| 第 2 周 | 场景适配 | "J 人旅行者为什么都在用圆周旅迹" |
| 第 3 周 | 使用教程 | "圆周旅迹怎么用？5 步完成第一个旅行计划" |
| 第 4 周 | 深度 Q&A | "关于圆周旅迹，用户问得最多的 8 个问题" |
| 第 5 周 | 竞品对比 | "圆周旅迹 vs 马蜂窝：自由行规划哪个更好用" |
| 第 6 周 | 用户故事 | "一个强迫症旅行者用圆周旅迹规划了 20 次出行" |
| 第 7 周 | 行业洞察 | "旅行规划工具进化史：为什么攻略平台满足不了年轻人" |

---

### 3.5 GEO 效果诊断

```
说：诊断GEO
```
输出一份测试 prompt 列表，去各 AI 平台手动测试：

测试方法：
1. 打开元宝 / DeepSeek / 豆包 / Kimi
2. 逐一输入诊断 prompt 里的问题
3. 记录：是否提到品牌名、描述是否准确、引用了哪篇文章

**建议节奏**：每发布 3-4 篇文章后做一次诊断，对比变化。

---

### 3.6 文章发布后

文章推送草稿箱后，你需要手动在微信公众号后台点"发布"。

发布后建议：
- 在社群/朋友圈引导真实阅读（阅读量影响 AI 的引用权重）
- 如果有用户对文章的反馈或提问，上传到问题库
```
说：上传问题：[用户反馈中的问题]
```

## 六、常见问题

**Q：文章生成了但没有推送到草稿箱？**
A：检查 WeWrite 是否已安装，以及 `config.yaml` 中是否填写了 `appid` 和 `secret`。
说"重新检查环境"让 skill 自动检测。

**Q：Q&A 模块里的问题感觉不够准确？**
A：上传你自己收集的真实用户问题（`上传问题`），skill 优先使用用户上传的问题，
比自动采集的更贴合你的实际用户。

**Q：同一类内容已经写了很多篇，但 AI 还是不引用？**
A：说"诊断GEO"检查当前 AI 平台的实际表现，找出具体问题。
常见原因：文章阅读量太低（AI 权重低）、brand_definition 写得太模糊、Q&A 问题和用户真实提问不匹配。

**Q：想换一个品牌用这个 skill？**
A：直接说"GEO设置"重新填写品牌信息，或者手动编辑 `geo-config.yaml`。
`query-bank.yaml` 和 `topic-bank.yaml` 建议同步清空重建。
