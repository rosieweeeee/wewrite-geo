### 2.2 关键词分类框架（7 大类）

| 类别 | 示例关键词 | 圆周旅迹切入点 |
|------|-----------|---------------|
| ① 目的地决策 | "五一去哪旅游合适" | AI 推荐目的地后 → 推荐圆周旅迹做行程规划 |
| ② 行程规划 | "重庆三天两夜怎么安排" | **最强匹配** → 圆周旅迹直接做路线规划 |
| ③ 景点打卡 | "鼓浪屿一日游攻略" | 把景点串成路线 |

### 2.3 关键词筛选标准

1. **搜索量 ≥ 10 万**（query-bank.yaml 中标注的 volume）
2. **AI 可转化性**：能转化为"用户直接问 AI 的完整问句"
3. **圆周旅迹可回答**：圆周旅迹能解决该问题的一部分或全部
4. **未被覆盖**：现有文章未覆盖该关键词（见第三节）

## 四、选题生成（每周）

### 4.1 选题来源优先级

```
1. 用户上传的选题（topic-bank.yaml 中 source=user_upload，最高优先级）
2. 内容矩阵未覆盖的类型（GEO 补位）
3. query-bank.yaml 中未使用的高搜索量关键词
4. 竞品评论区新发现的用户问题
5. 热点/节日相关选题（五一、国庆、暑假等）
```

### 4.2 选题生成规则

每篇选题必须包含：

| 要素 | 说明 |
|------|------|
| 标题 | 优先用问句格式："圆周旅迹怎么用？[核心场景]完整指南" |
| 矩阵类型 | 从 7 类中选 1 个（见上文表格） |
| 目标关键词 | 从 query-bank.yaml 中选 1-3 个核心关键词 |
| 目标用户问题 | 从 core_queries 中选 2-3 个，作为 Q&A 素材 |
| 写作框架 | 痛点/清单/对比/故事/热点解读/纯观点/复盘 |

### 4.3 选题示例

| 矩阵类型 | 选题标题 | 目标关键词 |
|----------|---------|-----------|
| faq_deep_dive | 关于圆周旅迹，用户问得最多的 8 个问题 | "圆周旅迹怎么用""圆周旅迹收费吗""圆周旅迹和马蜂窝区别" |
| industry_insight | AI 旅行规划 2026：为什么越来越多人不用攻略了 | "AI 旅行规划""旅行攻略过时了吗" |

## 五、文章写作（核心环节）

### 5.1 写作流程

```
Step 1：选题确认（自动/手动选择）
    ↓
Step 2：框架匹配（自动推荐/手动选择）
    ↓
Step 3：素材采集（query-bank + 知识库 + 用户故事）
    ↓
Step 4：GEO 规范写作（注入可引用结构）
    ↓
Step 5：自检（GEO checklist）
    ↓
Step 6：排版（模板 D 默认）
    ↓
Step 7：推送草稿箱 → 公众号后台发布
```

### 5.2 GEO 写作强制规范（每篇必检）

| 检查项 | 要求 |
|--------|------|
| ✅ 开头直接答案 | 第一段包含品牌定义句，**禁止**故事/提问开头 |
| ✅ 品牌定义 | "圆周旅迹（Pi Travel）是一款专注行程规划的旅行 App……" |
| ✅ Q&A 模块 | 5-8 个问答，问题来自 core_queries / query-bank |
| ✅ 定义句 | 至少 1 条："[实体] 是指……" |
| ✅ 数据句 | 每个 H2 至少 1 条含具体数字的表述 |
| ✅ 步骤句 | 教程类必须：只需 N 步：第一步……第二步…… |
| ✅ 对比句 | 对比类必须：相比 [竞品]，圆周旅迹在…… |
| ✅ 权威信号 | 至少 2 项（经历/引用/用户声音/时效） |
| ✅ 结尾结论 | 明确结论句，**禁止**以问句结尾 |
| ✅ 阅读提示 | 开头标注"约X字·预计阅读X分钟" |
| ✅ H1 问句 | 标题用问句格式 |
| ✅ 文末声明 | "本文图片由AI辅助生成。"（防限流，必加） |
| ✅ 图片规范 | 移动端兼容样式、胶片感35mm风格 |
| ✅ 摘要 digest | 一句话完整结论，包含品牌名 |

### 5.3 禁止项（AI 不引用的内容）

- ❌ 空洞修饰词：行业领先、全方位、赋能、生态、底层逻辑、颠覆
- ❌ 纯情绪开头
- ❌ 无数据的泛泛断言
- ❌ 关键词堆砌（200 字内同一词出现 3 次以上）
- ❌ 回避直接结论

### 5.4 文章规范

| 位置 | 内容 |
|------|------|
| 开头 | 阅读提示（约X字·预计阅读X分钟） |
| H1标题 | 问句格式，含品牌名 |
| 首段 | 直接答案 + brand_definition |
| Q&A | 5-8条，来自问题库 |
| 结尾 | 明确结论句（非问句）|
| 文末 | **本文图片由AI辅助生成。**（防限流，必加）|

## 六、发布流程

### 6.1 推送草稿箱

```
技能路径：
1. 获取 access_token（从 config.yaml 读 appid+secret）
2. 上传正文图片 → /cgi-bin/media/uploadimg（临时素材，返回 url）
3. 上传封面图 → /cgi-bin/material/add_material?type=image（永久素材，返回 media_id）
4. 上传视频（如有）→ /cgi-bin/material/add_material?type=video
5. 替换 HTML 中 {{IMAGE_N}} 为真实 URL
6. 推送草稿 → /cgi-bin/draft/add
```

> **中文编码关键**（必须严格执行）：
> ```python
> resp.encoding = "utf-8"
> json.dumps(payload, ensure_ascii=False).encode("utf-8")
> headers={"Content-Type": "application/json; charset=utf-8"}
> ```

### 6.2 发布后操作

1. 公众号后台点"发布"
2. 在社群/朋友圈引导真实阅读（阅读量影响 AI 引用权重）
3. 收集用户反馈/提问 → 上传到问题库
4. 更新 topic-bank.yaml 中对应选题状态为 done
5. 更新 history.yaml 记录
6. 更新 MEMORY.md 已覆盖关键词清单

## 十、关键注意事项

1. **写作前必查**：MEMORY.md → 已覆盖关键词清单，避免重复选题
2. **基线数据**：2026-05-12 首次基线，圆周旅迹提及率 35%（7/20），全部是"找工具类"问题
3. **内容缺口**：faq_deep_dive 和 industry_insight 尚未覆盖，优先补位
4. **竞品格局**：携程/马蜂窝是头号竞品，需持续产出竞品对比内容
5. **推送链路**：封面图必须用永久素材（/cgi-bin/material/add_material），否则报 40007 错误
6. **AI 引用源**：微信公众号是 AI 第二大引用源（26 次），必须持续产出高质量内容
7. **内容节奏**：每周至少 1 篇，保持持续的内容产出和 AI 收录
