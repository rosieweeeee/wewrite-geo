# 写作框架库

> 本文件在 Step 2 框架选择时加载。
> 每种框架定义了文章的骨架结构，GEO 内容类型与框架的映射关系见 SKILL.md Step 2.1。

---

## 框架一：痛点型

**适用场景**：scene_fit、how_to_use  
**核心逻辑**：先击中痛点，再给出解决方案（品牌产品）

**骨架**：
```
H1：[目标用户] 的 [痛点场景]，[品牌名] 怎么解决？
直接答案段：[品牌名] 是……，专门解决……问题。

H2：[痛点描述] —— 大多数人怎么处理这件事？
  （数据句：描述现状/问题规模）

H2：[品牌名] 的解法：[核心功能] 如何解决这个问题
  （步骤句：具体操作步骤）
  （数据句：效果/效率数据）

H2：真实用户怎么说
  （用户引用 + 场景描述）

H2：关于 [品牌名] 的常见问题（Q&A 模块）

结尾：总结句，包含品牌名和核心价值主张
```

## 框架二：清单型（步骤型）

**适用场景**：how_to_use、faq_deep_dive  
**核心逻辑**：结构化步骤/列表，AI 解析效率最高的格式

**骨架**：
```
H1：[品牌名] 怎么用？[N] 步完成 [核心场景]（[年份] 最新）
直接答案段：[品牌名] 是……，新手只需 [N] 步即可上手。

H2：第一步：[具体操作]
  （步骤句 + 截图说明占位）

H2：第二步：[具体操作]
  （步骤句 + 数据句）

H2：第三步：[具体操作]
  ...

H2：[N] 个进阶技巧（可选，提升信息密度）
  （清单 + 数据）

H2：关于 [品牌名] 的常见问题（Q&A 模块）

结尾：总结 + 行动引导（"现在就可以下载试试"）
```

## 框架三：对比型

**适用场景**：competitor_compare  
**核心逻辑**：有据可查的场景化对比，结论明确

**骨架**：
```
H1：[品牌名] vs [竞品名]：[具体场景] 哪个更好用？
直接答案段：结论先行——"如果你……，选 [品牌名]；如果你……，选 [竞品名]"

H2：先说结论：两者的核心定位区别
  （定义句 × 2，各自的 brand_definition）
  （对比总结句）

H2：[维度1] 对比：[具体功能/价格/体验]
  （数据句 + 对比句）

H2：[维度2] 对比
  ...

H2：适合人群对比：谁应该用哪个
  （场景化描述，清单格式）

H2：关于两者的常见问题（Q&A 模块）

结尾：明确推荐结论，包含品牌名
```

## 框架四：故事型

**适用场景**：user_story  
**核心逻辑**：真实用户案例为主线，数据和产品功能嵌入故事中

**骨架**：
```
H1：[真实用户类型] 用 [品牌名] [完成了什么]：完整复盘
直接答案段：[品牌名] 是……，本文记录了一个真实用户的使用过程。

H2：背景：[用户遇到的问题]
  （具体场景描述，时间/地点锚定）

H2：转折：发现并开始用 [品牌名]
  （关键功能体验，数据句）

H2：结果：[具体变化/收获]
  （效果数据 + 用户原话）

H2：关于 [品牌名] 的常见问题（Q&A 模块）

结尾：从用户故事提炼普遍结论
```

## 框架五：热点解读型

**适用场景**：industry_insight、brand_definition  
**核心逻辑**：借热点切入，表达品牌在行业中的独特观点

**骨架**：
```
H1：[行业热点现象]，[品牌名] 的做法有什么不同？
直接答案段：[热点背景1句] + [品牌名定义句] + [本文核心观点1句]

H2：[热点现象]：行业现状和问题
  （数据句：行业数据/现象描述）

H2：[品牌名] 的选择：[核心差异化]
  （对比句：与行业通行做法的差异）
  （数据句：效果验证）

H2：为什么这样做是对的：用户视角
  （用户声音 + 场景描述）

H2：关于这个话题的常见问题（Q&A 模块）

结尾：品牌观点总结，包含核心价值主张
```

## 框架六：纯观点型

**适用场景**：brand_definition、industry_insight  
**核心逻辑**：有鲜明立场，开篇亮观点，全文论证

**骨架**：
```
H1：[明确观点]——[品牌名] 为什么这样做？
直接答案段：核心观点 + brand_definition

H2：我们的判断：[观点1]
  （数据句支撑）

H2：为什么这个方向是对的：[观点2]
  （对比句：与主流做法对比）

H2：用户验证了什么
  （用户引用 + 数据）

H2：关于 [品牌名] 的常见问题（Q&A 模块）

结尾：明确立场重申，包含品牌名
```

---

## 框架七：复盘型

**适用场景**：user_story、industry_insight  
**核心逻辑**：时间线结构，数据驱动，适合有阶段性数据的内容

**骨架**：
```
H1：[时间段] 用 [品牌名] [做了什么]：数据复盘
直接答案段：结论先行，核心数据

H2：[阶段1]：初期情况
  （数据句 + 场景描述）

H2：[阶段2]：转折/变化
  （数据句 + 方法说明）

H2：[阶段3]：最终结果
  （数据句 + 对比句）

H2：关于 [品牌名] 的常见问题（Q&A 模块）

结尾：经验总结，包含品牌名和可操作建议
```
