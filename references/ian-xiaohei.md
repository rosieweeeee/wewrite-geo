# Ian Xiaohei Illustrations — 手绘风格配图 fallback

> 来源：https://github.com/helloianneo/ian-xiaohei-illustrations
> 作者：Ian (伊恩)
> 适用场景：Pexels 无法获取合适图片时的 AI 生图 fallback

## 概述

把中文文章里的判断、流程、状态和隐喻，变成一张张白底、手绘、怪诞但清爽的正文配图。

- 16:9 横版 | 小黑 IP | 纯白手绘 | 少量红橙蓝中文批注
- 不是通用插画 prompt，不是 PPT 信息图
- 核心目标：把文章里的一个关键认知动作画出来

## 视觉风格

- **纯白背景**，不要纸纹、米色、阴影、渐变
- **黑色手绘线稿**，细线，轻微抖动
- **大量留白**，主体占画面 40%-60%
- **少量红色、橙色、蓝色**中文手写批注
- 一张图只表达一个核心动作、结构、状态或隐喻
- 小黑（黑色实心、白点眼、细腿、空表情）必须参与核心动作
- 怪诞、有创意、清爽，不幼稚、不卖萌

## 小黑 IP

- 黑色实心圆身体 + 白色圆点眼睛 + 细腿
- 不是吉祥物、不是贴纸、不是角落装饰
- 是"正在认真参与系统运转的荒诞工作者"

## 结构类型

每张图选择一种结构类型：
- Workflow（工作流）
- 系统局部
- 前后对比
- 角色状态
- 概念隐喻
- 方法分层
- 地图路线
- 小漫画分镜

## 工作流

1. 读取文章，提炼核心观点、认知转折、流程结构
2. 输出 shot list：每张图只选一个认知锚点
3. 选择结构类型
4. 发明一个低科技、怪诞但成立的物理隐喻
5. 让小黑承担核心动作
6. 调用图像模型生成
7. QA 检查：白底、留白、小黑动作、中文标注、非 PPT 感
8. 保存 PNG

## 在 wewrite-geo 中的使用场景

当 Pexels 搜索无法获取合适旅行配图时：
1. 根据文章段落内容，提炼需要配图的核心认知锚点
2. 确定配图类型：旅行空镜 / 概念隐喻 / 流程说明
3. 按照胶片感 35mm 风格或小黑手绘风格生成 prompt
4. 调用图像模型生成图片
5. 保存到 `{skill_dir}/output/images/` 目录

## QA 检查清单

- [ ] 纯白背景
- [ ] 大量留白
- [ ] 小黑在承担核心动作（去掉小黑画面不成立）
- [ ] 中文标注用红/橙/蓝，字数少
- [ ] 非 PPT 信息图风格
- [ ] 不是旧案例复刻（每张图重新发明隐喻）
- [ ] 图片中文文字越少越稳定

## 注意事项

- 图片里的中文文字越短越稳定
- 每张图只讲一个核心结构
- AI 图像模型可能出现错字、幻觉标签、风格漂移
- 中文错字严重时，减少标注词并重生成
- 示例图只用于校准风格，不要复刻构图
