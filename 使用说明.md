## 一、第一次使用：安装准备

> ⚠️ 只需要做一次，做完之后每次发文章直接跳到第二步。

### 1.1 打开终端

**Mac 用户**：
1. 按键盘上的 `Command（⌘）+ 空格`，弹出搜索框
2. 输入 `终端`，按回车
3. 出现一个黑色（或白色）的窗口，这就是终端

### 1.2 安装 Python 依赖

在终端里复制粘贴以下命令，按回车执行：

```bash
pip3 install requests pyyaml
```

等出现 `Successfully installed` 字样就说明成功了。

> 如果提示 `pip3: command not found`，换成：
> ```bash
> python3 -m pip install requests pyyaml
> ```

### 1.3 配置微信公众号信息

你需要在一个配置文件里填入微信公众号的 AppID 和 AppSecret。

**第一步**：在终端执行以下命令，创建配置文件夹：

```bash
mkdir -p ~/.openclaw/skills/wewrite
```

**第二步**：创建配置文件：

```bash
cat > ~/.openclaw/skills/wewrite/config.yaml << 'EOF'
appid: "在这里填你的AppID"
secret: "在这里填你的AppSecret"
EOF
```

**第三步**：填入真实的 AppID 和 AppSecret。

去哪里找？登录 [微信公众平台](https://mp.weixin.qq.com) → 左侧菜单「设置与开发」→「基本配置」→ 找到 **AppID** 和 **AppSecret**（AppSecret 需要点「显示」并验证身份）。

**第四步**：编辑配置文件，把刚才找到的值填进去：

```bash
open ~/.openclaw/skills/wewrite/config.yaml
```

这会用文本编辑器打开文件，把 `在这里填你的AppID` 替换成真实的 AppID，保存即可。

### 1.4 把推送脚本放到方便的位置

把 `toolkit/push_draft.py` 这个文件复制到你的桌面：

```bash
cp /path/to/wewrite-geo/toolkit/push_draft.py ~/Desktop/push_draft.py
```

> 把 `/path/to/wewrite-geo/` 替换成你实际解压文件夹的路径。
> 比如你把文件夹放到了桌面，就是：
> ```bash
> cp ~/Desktop/wewrite-geo/toolkit/push_draft.py ~/Desktop/push_draft.py
> ```

## 二、每次发文章的流程

### 流程总览

```
AI 生成文章 HTML
    ↓
准备配图（5张，胶片感风格）
    ↓
在终端运行推送脚本
    ↓
去公众号草稿箱预览确认
    ↓
发布 ✅
```

---

### 2.1 准备文件

每篇文章需要准备：
- **1个 HTML 文件**（AI 生成，文件名如 `article01.html`）
- **5张图片**（你用即梦/Midjourney 生成的胶片感图片）
  - 图1 = 封面图（略偏暗的那张）
  - 图2~5 = 正文配图

把这些文件全部放到桌面的一个文件夹里，比如 `~/Desktop/article01/`。

### 2.2 打开终端

按 `Command（⌘）+ 空格` → 搜索「终端」→ 回车。

### 2.3 运行推送脚本

在终端里输入以下命令（根据实际文件名修改）：

```bash
python3 ~/Desktop/push_draft.py \
  --html ~/Desktop/article01/index.html \
  --title "我用过几十个旅行App，最后只留下了这一个" \
  --images ~/Desktop/article01/图1.jpg \
           ~/Desktop/article01/图2.jpg \
           ~/Desktop/article01/图3.jpg \
           ~/Desktop/article01/图4.jpg \
           ~/Desktop/article01/图5.jpg \
  --cover ~/Desktop/article01/图1.jpg
```

**参数说明**：

| 参数 | 说明 | 示例 |
|------|------|------|
| `--html` | HTML 文件路径 | `~/Desktop/article01/index.html` |
| `--title` | 文章标题（用引号括起来） | `"文章标题"` |
| `--images` | 正文图片，按顺序列出 | `图1.jpg 图2.jpg ...` |
| `--cover` | 封面图（通常是图1） | `~/Desktop/article01/图1.jpg` |
| `--digest` | 文章摘要（可选，有默认值） | `"一句话描述"` |

### 2.4 看结果

运行成功后终端会显示：

```
✅ Token 获取成功
✅ 正文图片上传成功：图1.jpg
✅ 正文图片上传成功：图2.jpg
...
✅ 封面图上传成功：图1.jpg
📤 推送草稿：文章标题
🎉 推送成功！草稿 media_id：xxxxxxxx
   请前往公众号后台 → 草稿箱 查看
```

看到 `🎉 推送成功` 就完成了！

### 2.5 在公众号后台确认

1. 打开 [微信公众平台](https://mp.weixin.qq.com)
2. 左侧菜单「内容与互动」→「草稿箱」
3. 找到刚推送的文章，点「预览」在手机上查看效果
4. 没问题就点「发布」

## 三、常见问题

### ❓ 终端显示"找不到文件"

检查文件路径是否正确。可以直接把文件拖到终端窗口里，会自动填入完整路径。

### ❓ 报错 `40007 invalid media_id`

封面图上传有问题。确认 `--cover` 参数指向了正确的图片文件。

### ❓ 草稿箱里文字是乱码

这是脚本版本问题，请确认使用的是最新版 `push_draft.py`（2026-04-23 版本）。

### ❓ 报错 `Connection refused` 或网络超时

检查网络连接是否正常，微信 API 需要能访问外网。

### ❓ 配图要用什么风格？

统一使用 **35mm胶片感** 风格，用即梦、Midjourney 或其他 AI 绘图工具生成。

Prompt 格式：
```
35mm film photography style, [场景描述], film grain texture, warm faded tones, 
analog film scan quality, no text
```

内容只拍旅行场景：空镜、人物背影、街道、自然景色。
**不要** 出现手机截图或 App 界面。

封面图（图1）加一句 `slightly underexposed`（让图偏暗，方便叠标题文字）。

### ❓ 有视频要怎么加？

在推送命令里加 `--videos` 参数：

```bash
python3 ~/Desktop/push_draft.py \
  --html ~/Desktop/article01/index.html \
  --title "文章标题" \
  --images 图1.jpg 图2.jpg 图3.jpg 图4.jpg 图5.jpg \
  --cover 图1.jpg \
  --videos ~/Desktop/产品演示1.mp4 ~/Desktop/产品演示2.mp4
```

视频会自动上传并嵌入到文章中对应的位置（HTML 里需要有 `{{VIDEO_1}}` 占位符）。
